Resumen y finalidad

La clasificación del hardware organiza los componentes físicos de un sistema informático según criterios prácticos: función, permanencia, ubicación, propósito y arquitectura. Esta página explica esos criterios, muestra ejemplos de cada categoría y ofrece actividades interactivas para practicar la clasificación y afianzar conceptos.

Clasificación por función

El criterio más intuitivo agrupa el hardware según la función que realiza dentro del sistema:

  • Entrada: dispositivos que permiten introducir datos (teclado, ratón, escáner, micrófono, sensores).
  • Procesamiento: componentes que ejecutan instrucciones y calculan (CPU, GPU, DSP, FPGAs).
  • Almacenamiento: elementos que guardan información de forma temporal o permanente (RAM, SSD, HDD).
  • Salida: dispositivos que presentan resultados (monitor, impresora, altavoces, actuadores).
  • Comunicación: elementos que permiten conectar sistemas (tarjeta de red, módem, router, antenas)

Cada categoría puede subdividirse: por ejemplo, almacenamiento en volatile (RAM) y no volatile (SSD/HDD).

Clasificación por permanencia

Este criterio distingue hardware según la duración de los datos que guarda:

  • Volátil: pierde su contenido cuando se corta la energía (RAM).
  • No volátil: conserva la información sin energía (SSDs, HDDs, EEPROM).

Este aspecto es crítico para diseñar sistemas: memoria volátil favorece velocidad; memoria no volátil favorece persistencia.

Clasificación por ubicación / forma

Según su ubicación física y montaje:

  • Interno: integrado dentro del chasis: placa base, CPU, RAM, discos internos.
  • Externo: periféricos conectados por puertos o inalámbricamente: impresoras, teclados, cámaras.
  • Embebido: hardware integrado en equipos no exclusivamente informáticos (microcontroladores en electrodomésticos, automóviles).

Clasificación por propósito / alcance

Aquí se distingue según la finalidad:

  • Propósito general: PC, servidores, dispositivos móviles — diseñados para múltiples usos.
  • Propósito específico (embebido): controladores, PLCs, sistemas de tiempo real para funciones concretas.
  • HPC y servidores: hardware optimizado para alto rendimiento y escalabilidad (clusters, racks, aceleradores).

Clasificación por arquitectura

Este criterio atiende a la organización lógica del hardware:

  • Von Neumann: memoria y programa compartido; CPU secuencial (la arquitectura clásica).
  • Harvard: memoria separada para instrucciones y datos (útil en microcontroladores).
  • Heterogénea: sistemas con CPU + GPU + NPU/TPU que reparten tareas según paralelismo y tipo de cálculo.

Ejemplos prácticos y correspondencia entre criterios

Un mismo componente puede pertenecer a múltiples clasificaciones: por ejemplo, una tarjeta SD es almacenamiento (por función), no volátil (por permanencia), y externa (por ubicación). Un microcontrolador es proc. y suele usarse en sistemas embebidos (propósito específico) con arquitectura Harvard.
